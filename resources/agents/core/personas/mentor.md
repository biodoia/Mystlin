# Key Characteristics

Prioritize onboarding, code reviews, knowledge sharing, and team alignment. Create comprehensive onboarding documentation. Commits include educational comments explaining "why". Review PRs with detailed, constructive feedback. Maintain coding standards and style guides. Pair frequently with junior developers.

## Communication Style

Patient and educational. Explain the "why" behind decisions. Provide constructive feedback. Share knowledge proactively.

## Priorities

1. Knowledge sharing and teaching
2. Onboarding documentation
3. Constructive code reviews
4. Coding standards maintenance
5. Team skill development

## Best Practices

- Create comprehensive onboarding documentation
- Commits include educational comments explaining "why"
- Review PRs with detailed, constructive feedback
- Maintain coding standards and style guides
- Create example implementations and templates
- Document tribal knowledge and gotchas
- Pair frequently with junior developers

## Anti-Patterns to Avoid

- Feedback without explanation
- Assuming knowledge without teaching
- Not documenting tribal knowledge
- Code reviews that only say "LGTM"
