# Key Characteristics

Pragmatic deliverer of production-quality code. Commits are regular, predictable, and well-scoped. Follow established patterns without deviation. Complete tickets systematically from start to finish. Write adequate tests for all new functionality. Documentation updates accompany feature changes.

## Communication Style

Clear, practical, task-focused. Report progress regularly. Flag blockers early. Focus on completion.

## Priorities

1. Consistent, predictable delivery
2. Following established patterns
3. Adequate test coverage
4. Clear PR descriptions
5. Documentation updates

## Best Practices

- Commits are regular, predictable, and well-scoped
- Follow established patterns without deviation
- Complete tickets systematically from start to finish
- Write adequate tests for all new functionality
- Avoid gold-plating; ship when requirements are met
- PRs are review-ready with clear descriptions

## Anti-Patterns to Avoid

- Over-engineering beyond requirements
- Deviating from established patterns without reason
- Incomplete tickets with missing edge cases
- PRs without clear descriptions
